# Product UI Specification Writer

## Description

用于编写项目产品 UI 规范、设计约束、页面风格说明和组件视觉规则的 Skill 模板。
当用户提到“UI 规范”、“设计规范”、“产品视觉标准”、“页面设计规则”、“组件风格统一”时激活。

## Usage Intent

这个 Skill 不是直接产出页面代码，而是先把产品 UI 规则整理为一份可复用的规范文档，方便后续：

- 页面设计统一
- 组件风格统一
- 新需求按统一标准落地
- 设计稿和开发实现对齐
- 多人协作时减少风格漂移

## Output Goal

产出内容应优先是“规范文档”而不是零散建议。

规范文档建议包含以下结构：

1. 产品概述
2. 品牌与视觉关键词
3. 色彩系统
4. 字体系统
5. 间距与圆角规范
6. 阴影与边框规范
7. 图标与插画规范
8. 布局规则
9. 页面层级规范
10. 表单规范
11. 按钮规范
12. 表格/列表规范
13. 状态规范
14. 动效规范
15. 响应式规范
16. 禁止事项
17. 页面落地检查清单

## Writing Rules

- 必须写成“可执行规范”，避免空泛描述
- 每条规范尽量包含：用途、规则、推荐值、禁止事项
- 如果用户没有给具体品牌信息，先使用占位字段输出，等待用户替换
- 优先输出结构化标题与条目，不要只写一段大而泛的说明
- 如果用户只说“写一份 UI 规范”，默认按 Web 管理后台/内容平台场景来写
- 如果用户项目已有视觉风格，规范必须继承现有风格，而不是推翻重来
- 规范要兼顾设计和开发落地，必要时补充 token 命名建议

规则：

- 页面信息从上到下遵循“概览 -> 重点 -> 详细内容”
- 卡片区块要有清晰的主次层级
- 避免页面元素过度拥挤或无明显视觉锚点

### 9. 页面层级规范

- 一级页面：用于承载完整业务模块
- 二级页面：用于功能详情、管理页、分析页
- 三级内容：用于卡片、局部面板、浮层内容

规则：

- 一级页面标题必须明确表达页面目的
- 二级区域要有模块标题或语义分组
- 不要出现多个同权重视觉中心相互竞争

### 10. 表单规范

- 输入框高度：`[如 40 / 44 / 48]`
- 标签位置：`[上方 / 左侧]`
- 必填标识：`[在此替换]`
- 错误提示样式：`[在此替换]`

规则：

- 表单错误提示优先就近展示
- 必填、禁用、只读、错误、成功状态必须有明确区分
- 长表单必须进行视觉分组

### 11. 按钮规范

按钮类型建议：

- 主按钮：核心动作
- 次按钮：辅助动作
- 线框按钮：弱操作
- 文字按钮：最低优先级操作
- 危险按钮：删除/不可逆动作

按钮规则：

- 一个操作区主按钮通常只保留 `1` 个
- 高风险操作按钮不要与主按钮视觉过于接近
- 按钮文案必须使用清晰动作词

### 12. 表格 / 列表规范

- 表头风格：`[在此替换]`
- 行高：`[在此替换]`
- 斑马纹：`[开 / 关]`
- Hover 效果：`[在此替换]`

规则：

- 数据密集界面优先保证可读性而不是装饰性
- 列表操作按钮不宜过多
- 表格中的状态色与全局状态色保持一致

### 13. 状态规范

必须定义的状态：

- 默认
- Hover
- Active
- Focus
- Disabled
- Loading
- Empty
- Error
- Success

规则：

- 每个核心组件至少要定义默认、悬停、禁用、激活四种状态
- 空状态必须提供解释文案和引导动作
- 错误状态必须可定位、可恢复

### 14. 动效规范

- 页面切换时长：`[如 200ms / 300ms]`
- Hover 动效时长：`[如 150ms / 200ms]`
- 弹窗出现时长：`[如 220ms / 260ms]`

规则：

- 动效用于说明层级变化和反馈，不是炫技
- 避免大幅晃动、频繁缩放、无意义连续动画
- 页面主要动效风格要统一

### 15. 响应式规范

- 桌面端断点：`[在此替换]`
- 平板断点：`[在此替换]`
- 移动端断点：`[在此替换]`

规则：

- 优先保证核心功能在小屏上可用
- 卡片、表格、筛选区在小屏需要重排策略
- 不允许仅缩放页面来假装适配移动端

### 16. 禁止事项

- 禁止滥用高饱和颜色
- 禁止使用风格不一致的图标
- 禁止页面层级混乱、标题权重不统一
- 禁止按钮类型过多导致主次不明
- 禁止无意义装饰遮挡内容可读性
- 禁止只给视觉描述，不给尺寸/颜色/状态规则

### 17. 页面落地检查清单

每次页面设计或开发完成后，至少检查：

- 是否符合品牌关键词
- 是否遵循色彩系统
- 是否使用统一字号和字重
- 是否遵循统一间距与圆角
- 是否存在明确主次层级
- 是否补齐交互状态
- 是否兼容桌面与移动端
- 是否避免风格漂移

## Optional Output Modes

如果用户指定目标，可以切换输出方式：

- **完整规范版**：输出完整 UI 规范文档
- **精简规则版**：输出一页式设计约束清单
- **Token 版**：输出为设计 token / CSS 变量结构
- **页面评审版**：按规范检查现有页面是否合规
- **组件规范版**：只输出按钮、输入框、卡片、表格等组件规则

## Customization Notes

你可以自行替换以下占位信息：

- 品牌名
- 主题色
- 辅助色
- 字体
- 圆角
- 阴影
- 页面宽度
- 动效时长
- 组件尺寸
- 响应式断点

如果没有给具体内容，默认保留为占位字段，方便后续直接替换。
